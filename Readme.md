An end-to-end Python pipeline for processing medical `.dcm` (DICOM) files. This toolkit provides utilities to anonymize Protected Health Information (PHI), verify the scrubbing process, convert images to standard `.png` formats, and leverage Google's Gemini AI to generate structured, step-by-step clinical analysis reports.

## ✨ Features

### 🛡️ DICOM Anonymization & Verification (`Dicomanon/`)
- **PHI Scrubbing:** Automatically clears or overrides standard Protected Health Information (PHI) tags (e.g., Patient Name, ID, DOB, Institution).
- **Private Tag Removal:** Strips proprietary and potentially identifying private tags generated by medical equipment.
- **Verification Tool:** Compares original and anonymized files side-by-side in a terminal table to guarantee data safety.

### 🖼️ Automated DICOM Conversion
- **Batch Processing:** Recursively scans directories to convert `.dcm` files into `.png` images.
- **3D Volume Support:** Automatically detects multi-frame/3D DICOMs and extracts them as grouped, individual frames.
- **Image Optimization:** Applies VOI LUT (Value of Interest Look-Up Table) and Photometric Interpretations (e.g., MONOCHROME1 inversion) for optimal visual contrast.

### 🤖 AI-Powered Medical Analysis
- **Advanced AI Reasoning:** Utilizes Google's Gemini AI (with the *Thinking* configuration) to emulate step-by-step diagnostic reasoning.
- **Grouped Context:** Analyzes multiple frames belonging to the same original DICOM file together for comparative analysis.
- **Structured Reporting:** Outputs detailed Markdown (`.md`) reports including the AI's internal thought process, visual findings, clinical impressions, and actionable recommendations.
- **CLI Parameter Control:** Complete control over inputs, outputs, models, and API keys through command-line flags.
